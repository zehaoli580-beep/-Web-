# 《基于Web的高校图书管理网站》—— 功能列表

> 说明：本文档作为功能需求起点，后续将以此为基础编写接口文档。
> 技术栈：Vue 3 + Node.js (Express) + MongoDB
> 开发周期：7天 | 协作方式：Git 版本控制 + 分工

---

## 一、用户模块

### 1.1 用户注册
- 学生/教师使用学号/工号注册
- 填写基本信息：姓名、学院、联系方式（邮箱/手机）
- 密码加密存储（bcrypt）
- 注册后需管理员审核激活（或自动激活）

### 1.2 用户登录
- 学号/工号 + 密码登录
- 记住密码（Token 持久化）
- 验证码校验（可选，可降低开发复杂度先不做）

### 1.3 用户角色与权限
| 角色 | 权限 |
|------|------|
| **普通用户（学生/教师）** | 查询图书、借阅、归还、查看个人借阅历史、修改个人信息 |
| **管理员** | 以上权限 + 图书管理（增删改）、用户管理、借阅审批/确认、统计查看 |

### 1.4 个人信息管理
- 查看/修改个人资料（姓名、联系方式、密码修改）
- 查看个人借阅记录（当前借阅 + 历史借阅）
- 查看个人违规/逾期记录

---

## 二、图书管理模块

### 2.1 图书录入
- 录入新图书信息：
  - ISBN（国际标准书号）
  - 书名
  - 作者
  - 出版社
  - 出版年份
  - 图书分类（文学、科学、工程、艺术……）
  - 馆藏数量 / 可借数量
  - 馆藏位置（楼层/书架号）
  - 封面图片（URL或上传）
  - 简介/目录

### 2.2 图书编辑
- 修改图书信息（管理员）
- 调整馆藏数量

### 2.3 图书下架/删除
- 下架已损坏或丢失的图书
- 仅可删除从未被借阅过的图书（维护数据完整性）

### 2.4 图书状态
- 在馆 / 已借出 / 下架
- 实时更新可借数量

---

## 三、借阅管理模块

### 3.1 借书
- 用户检索图书并发起借阅申请
- 管理员确认借出，系统自动记录：
  - 借书人、图书、借书日期
  - 应还日期（默认30天，可配置）
- 生成借阅记录

### 3.2 还书
- 管理员确认归还
- 系统自动计算是否逾期
- 若逾期则生成逾期记录及罚款金额（可选）
- 更新图书状态为"在馆"

### 3.3 续借
- 用户在应还日期前可申请续借一次
- 续借延长15天（或原借期的一半）
- 已被其他用户预约的图书不可续借

### 3.4 预约
- 当图书全部借出时，用户可预约
- 图书归还后按预约顺序通知用户（系统消息中心）
- 通知后保留24小时，超时自动释放给下一个预约者

### 3.5 逾期处理
- 逾期每天罚款 0.1 元（可配置）
- 逾期记录纳入用户信用
- 逾期超过60天将暂停借书权限

## 四、检索与浏览模块

### 4.1 图书检索
- 关键词检索：书名、作者、出版社、ISBN
- 模糊匹配（正则/全文索引）
- 检索结果分页展示

### 4.2 高级筛选
- 按分类筛选
- 按出版年份范围
- 按馆藏状态（在馆/全部）

### 4.3 图书详情页
- 展示全部图书信息
- 展示当前借阅状态
- 展示借阅历史（对该书的借阅次数）
- 展示相关图书推荐（同分类/同作者）

### 4.4 热门/新书推荐
- 首页展示热门借阅图书排行榜
- 展示最新入库图书

---

## 五、消息中心

### 5.1 消息列表
- 展示当前用户的所有系统消息
- 支持已读/未读状态切换
- 支持标记全部已读
- 分页展示，按时间倒序

### 5.2 消息类型
- **预约到书通知**：预约的图书已归还，通知用户来借阅
- **图书逾期提醒**：借阅的图书已逾期，提醒用户尽快归还
- **图书到期提醒**：借阅图书临近应还日期（提前3天），提醒用户及时归还或续借

### 5.3 消息生成时机
- 管理员确认归还时 → 检查预约队列 → 向预约用户发送"预约到书通知"
- 用户登录时 → 检查是否有逾期或即将到期的借阅 → 自动生成提醒消息

## 六、统计与数据看板（管理员）

### 6.1 图书统计
- 馆藏总量、分类分布
- 借出/在馆比例

### 6.2 借阅统计
- 日/周/月借阅量
- 热门图书排行榜
- 热门分类排行榜

### 6.3 用户统计
- 注册用户总数
- 活跃借阅用户数

### 6.4 逾期统计
- 逾期率
- 总罚款金额

---

## 七、系统管理（管理员）

### 7.1 用户管理
- 查看所有用户列表
- 激活/禁用用户账号
- 重置用户密码

### 7.2 分类管理
- 新增/编辑/删除图书分类

### 7.3 参数配置
- 最大借阅数量（默认5本/人）
- 借阅天数（默认30天）
- 续借天数（默认15天）
- 逾期罚款金额（默认0.1元/天）

---

## 八、API 风格约定（为接口文档锚定基调）

- 基础路径：`/api/v1`
- RESTful 风格
- 请求体格式：JSON
- 响应统一格式：
  ```json
  {
    "code": 200,
    "message": "success",
    "data": { ... }
  }
  ```
- 认证方式：JWT（存入 localStorage，请求头携带 `Authorization: Bearer <token>`）
