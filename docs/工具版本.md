# 《基于Web的高校图书管理网站》—— 开发环境任务书

> 本文档是发给组员的环境配置指南。请每位组员按照以下要求安装和配置开发环境，**开始写代码前务必完成**，避免因环境不一致导致代码跑不起来。
>
> ⚠️ **版本必须统一**：我和我的电脑已安装 Node.js v24.15.0 和 MongoDB 8.3.2，所有人的版本必须与这个一致。

---

## 一、必须安装的软件

以下三项 **每个人都需要安装**，安装完后在命令行验证版本是否正确：

| 软件 | 版本要求 | 用途 | 验证命令 |
|:----|:--------:|:------|:---------|
| **Node.js** | **v24.15.0** | JavaScript 运行时（后端 + 前端构建工具都依赖它） | `node -v` |
| **npm** | **11.12.1** | 包管理器（装 Node.js 时自动附带） | `npm -v` |
| **MongoDB** | **8.3.2** | 数据库 | `mongod --version` |
| **Git** | 最新版 | 版本控制与分工协作 | `git --version` |
| **VS Code** | 最新版 | 代码编辑器 | — |

> 重点：**Node.js 和 MongoDB 版本必须和我一致**，否则可能出现兼容性问题，debug 起来很麻烦。

### 安装注意事项

- **Node.js**：官网下载 LTS 版本。我的是 v24.15.0，请安装同版本。
- **MongoDB**：装完后需要启动服务（Windows 搜索"服务" → 找到 MongoDB → 启动），或者命令行运行 `net start MongoDB`
- **Git**：装完后配置用户名和邮箱：`git config --global user.name "你的名字"`、`git config --global user.email "你的邮箱"`

### VS Code 推荐插件

在 VS Code 扩展商店搜索安装：

| 插件名 | 用途 |
|:------|:------|
| **Vue - Official** | Vue 3 语法高亮和智能提示 |
| **ESLint** | 代码检查 |
| **Prettier** | 代码格式化 |

## 二、项目会用到的 npm 包

以下包不需要手动下载，项目代码写好后在项目目录下运行 `npm install` 会自动安装。这里列出来是让你们知道项目用了什么：

### 2.1 前端（Vue 3 + Vite）

| 包名 | 版本 | 用途 |
|:----|:----:|:------|
| `vue` | ^3.5.35 | Vue 3 前端框架 |
| `vite` | ^6.3.5 | 构建工具（替代 vue-cli） |
| `vue-router` | ^4.5.1 | 前端路由（页面跳转） |
| `pinia` | ^3.0.4 | 状态管理（组件之间共享数据） |
| `axios` | ^1.17.0 | 发 HTTP 请求调后端接口 |
| `element-plus` | ^2.9.0 | UI 组件库（表格、表单、弹窗、分页开箱即用） |
| `echarts` | ^5.6.0 | 图表库（统计看板用） |

> Element Plus 强烈推荐给新手，不需要手写 CSS，拖组件就能搭出好看的页面。

### 2.2 后端（Express + Mongoose）

| 包名 | 版本 | 用途 |
|:----|:----:|:------|
| `express` | ^4.18.2 | Node.js Web 框架（写接口） |
| `mongoose` | ^9.6.3 | 连接和操作 MongoDB |
| `cors` | ^2.8.6 | 允许前端跨域请求后端 |
| `jsonwebtoken` | ^9.0.3 | JWT 登录令牌 |
| `bcryptjs` | ^3.0.3 | 密码加密（纯 JS，装起来没坑） |
| `nodemon` | ^3.1.0 | 改代码后自动重启后端（开发辅助） |
| `dotenv` | ^16.4.0 | 管理环境变量 |

## 三、项目初始化命令（组长建好仓库后共用）

以下命令组长建好项目仓库后，组员 clone 下来只需跑一次：

### 后端

```bash
cd server
npm install
```

### 前端

```bash
cd client
npm install
```

> 注意：不需要每个人单独跑 `npm create vite`，也不需要 `npm init -y`。这些由组长初始化好提交到 Git，其他人 clone 后直接 `npm install` 即可。

## 四、日常启动命令

### 第一步：启动 MongoDB

```bash
net start MongoDB
```

> 或者从 Windows 服务管理器启动。每次写代码前需要确保 MongoDB 在运行。

### 第二步：启动后端

```bash
cd server
npm run dev
```

> 后端默认跑在 `http://localhost:3000`

### 第三步：启动前端

```bash
cd client
npm run dev
```

> 前端默认跑在 `http://localhost:5173`，浏览器打开这个地址就能看到页面

### 注意事项

- 每次开机后第一次写代码，**必须按顺序**：启动 MongoDB → 启动后端 → 启动前端
- 三个终端窗口都要开着（MongoDB / 后端 / 前端），关掉任何一个项目就会停
- 后端代码修改后会自动重启（nodemon），前端代码修改后浏览器会自动刷新（vite）

## 五、如何确认环境配置成功

打开命令行，依次运行以下命令，每行都应该输出版本号而不是报错：

```bash
node -v          # 应输出 v24.15.0
npm -v           # 应输出 11.12.1
git --version    # 应输出 git version 2.xx
```

MongoDB 验证方法：打开命令行运行 `mongosh`，能连上就说明 MongoDB 启动成功。
